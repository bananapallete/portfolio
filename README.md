# Unlimit_Cho Portfolio

브랜딩 / 일러스트 / 모션그래픽을 소개하는 개인 포트폴리오 사이트입니다.

## 파일 구성

- `index.html`, `script.js` — 고객에게 보여줄 공개 페이지
- `admin.html`, `admin.js` — 콘텐츠를 수정하는 관리자 페이지 (공개 페이지 하단 "관리자용 편집 페이지" 링크로 이동 가능)
- `data.json` — 프로필/카테고리/프로젝트 데이터 (두 페이지가 공유)
- `style.css` — 공통 디자인

## 로컬에서 확인하기

브라우저로 `index.html`을 더블클릭해서 바로 열어도 되지만, 일부 브라우저는 보안 정책상 `file://` 경로에서 `data.json`을 못 읽어올 수 있습니다. 이 경우 두 페이지 모두 파일 선택창이 나타나 `data.json`을 직접 선택하면 됩니다.

더 편하게 보려면 이 폴더에서 로컬 서버를 띄우세요.

```bash
python -m http.server 8000
```

그다음 브라우저에서 `http://localhost:8000` 접속.

## 콘텐츠 수정 워크플로

1. `admin.html`을 열어 프로필, 프로젝트별 텍스트·이미지·영상을 수정합니다. 수정 내용은 브라우저에 자동 임시저장됩니다(새로고침해도 유지).
2. 상단 **미리보기** 버튼으로 실제 반영될 모습을 바로 확인할 수 있습니다.
3. 다 됐으면 상단 **내보내기(다운로드)** 버튼을 눌러 새 `data.json` 파일을 받습니다.
4. 받은 파일로 이 폴더 안의 `data.json`을 덮어씁니다.
5. 실제 사이트에 반영하려면 배포 중인 곳(GitHub Pages 등)에 다시 업로드/커밋 & 푸시합니다.

영상은 유튜브·비메오 링크를 추가하는 걸 추천합니다(파일 업로드도 가능하지만 용량이 크면 `data.json`이 무거워져 로딩이 느려질 수 있어요).

## GitHub Pages로 배포하기 (무료)

1. GitHub에 새 저장소를 만들고 이 폴더 안 파일 전체(`index.html`, `admin.html`, `style.css`, `script.js`, `admin.js`, `data.json`)를 업로드합니다.
2. 저장소 **Settings → Pages**로 이동, Branch를 `main`(또는 `master`), 폴더는 `/root`로 설정 후 저장합니다.
3. 몇 분 뒤 `https://<사용자이름>.github.io/<저장소이름>/` 주소로 사이트가 열립니다.
4. 이후 콘텐츠를 바꾸고 싶으면 위 "콘텐츠 수정 워크플로"대로 `data.json`을 다시 커밋 & 푸시하면 됩니다.

`admin.html`은 별도 로그인 기능이 없습니다. 누구나 URL을 알면 편집 화면을 열어볼 수는 있지만, 수정 내용이 실제 사이트에 반영되려면 파일을 직접 다운로드해 저장소에 다시 올려야 하므로 실수로 사이트가 바뀌는 일은 없습니다. 다만 편집 화면 자체를 남에게 보이고 싶지 않다면 `admin.html`의 URL을 따로 공유하지 않는 것을 추천합니다.
